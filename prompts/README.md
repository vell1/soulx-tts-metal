# AI 播客脚本生成提示词使用指南

本目录包含一套完整的 AI 提示词模板，帮助你使用 ChatGPT、Claude 等大语言模型将对话内容快速转换为符合 SoulX-TTS 系统格式的播客脚本。

---

## 📁 文件结构

```
prompts/
├── README.md                           # 本文档
├── podcast_generation_prompt.md        # 主提示词模板（完整功能）
└── scenarios/                          # 场景化提示词
    ├── format_dialogue.md              # 对话格式化
    ├── add_emotions.md                 # 情感标签添加
    └── multi_dialect.md                # 多方言脚本生成
```

## 🚀 快速开始

### 方式 1：使用完整提示词（推荐）

1. 打开 [podcast_generation_prompt.md](podcast_generation_prompt.md)
2. 复制"系统角色"到"注意事项"部分的全部内容
3. 粘贴到 ChatGPT/Claude 对话开头
4. 然后提供你的对话内容
5. AI 会自动生成符合格式的脚本

**示例对话**:
```
[粘贴完整提示词]

用户：请帮我格式化以下对话：
A: 大家好，欢迎来到今天的节目。
B: 你好，很高兴来到这里。
A: 今天我们要聊聊人工智能的话题。
B: 好的，这是一个很有趣的话题。

AI：[生成符合格式的播客脚本]
```

### 方式 2：使用简化提示（快速）

直接向 AI 发送：

```
请将以下对话转换为 SoulX-TTS 播客脚本格式：

格式要求：
1. 使用 @角色: 名称, 声音, 方言 定义角色
   - 可用声音：女声1、男声1
   - 可用方言：普通话、四川话、河南话、粤语
2. 使用 [角色名称]: 对话内容 格式化对话
3. 可以添加情感标签：
   - <|laughter|> 笑声
   - <|sigh|> 叹气
   - <|breathing|> 呼吸声
   - <|coughing|> 咳嗽

对话内容：
[在此粘贴你的对话]
```

### 方式 3：使用场景化提示词

根据具体需求选择：

| 需求 | 使用提示词 | 适用场景 |
|------|-----------|----------|
| 格式化对话 | [format_dialogue.md](scenarios/format_dialogue.md) | 已有对话，需要添加格式 |
| 添加情感 | [add_emotions.md](scenarios/add_emotions.md) | 脚本已格式化，需要增强情感 |
| 多方言混合 | [multi_dialect.md](scenarios/multi_dialect.md) | 需要多个方言的播客 |

## 💡 使用示例

### 示例 1：基础对话转换

**输入**（向 AI 发送）:
```
[粘贴主提示词]

请格式化以下对话：
主持人：大家好，欢迎来到今天的科技播客。
嘉宾：你好，很高兴来到这里。
主持人：今天我们要聊聊人工智能。
嘉宾：好的，这个话题很有意思。
```

**AI 输出**:
```
# 科技播客

# 角色定义
@角色: 主持人, 女声1, 普通话
@角色: 嘉宾, 男声1, 普通话

# 对话内容
[主持人]: 大家好，欢迎来到今天的科技播客。<|laughter|>
[嘉宾]: 你好，很高兴来到这里。
[主持人]: 今天我们要聊聊人工智能的话题。
[嘉宾]: 好的，这个话题很有意思。
```

### 示例 2：添加情感标签

**输入**:
```
[粘贴情感标签提示词]

请为以下脚本添加情感标签：

@角色: 主持人, 女声1, 普通话
@角色: 创业者, 男声1, 普通话

[主持人]: 创业这几年辛苦吗？
[创业者]: 确实很不容易。
[主持人]: 但你坚持下来了。
[创业者]: 是的，现在回想起来，都值得。
```

**AI 输出**:
```
@角色: 主持人, 女声1, 普通话
@角色: 创业者, 男声1, 普通话

[主持人]: 创业这几年辛苦吗？
[创业者]: <|sigh|> 确实很不容易。
[主持人]: 但你坚持下来了。<|laughter|>
[创业者]: 是的，<|sigh|> 现在回想起来，都值得。
```

### 示例 3：多方言播客

**输入**:
```
[粘贴多方言提示词]

请生成一个多方言播客脚本，主题是各地美食：
- 主持人用普通话
- 成都嘉宾用四川话
- 广州嘉宾用粤语
```

**AI 输出**:
```
# 美食文化交流

# 角色定义
@角色: 主持人, 女声1, 普通话
@角色: 成都嘉宾, 男声1, 四川话
@角色: 广州嘉宾, 女声1, 粤语

# 对话内容
[主持人]: 今天我们聊聊各地的美食文化。
[成都嘉宾]: 说到美食，我们成都的火锅那是巴适得很！<|laughter|>
[广州嘉宾]: 我哋广东就钟意饮早茶，一盅两件，好写意。
[主持人]: 听起来各有特色呢。<|laughter|>
```

## 📝 完整工作流程

### 步骤 1：准备对话内容

整理你的对话内容，可以是：
- 已有的文字对话
- 会议纪要
- 访谈记录
- 自己编写的对话大纲

### 步骤 2：选择合适的提示词

根据需求选择：
- 从零开始 → 使用主提示词
- 只需格式化 → 使用对话格式化提示词
- 需要多方言 → 使用多方言提示词
- 需要更生动 → 使用情感标签提示词

### 步骤 3：与 AI 对话

1. 打开 ChatGPT/Claude
2. 粘贴选择的提示词
3. 提供你的对话内容
4. AI 生成格式化脚本

### 步骤 4：检查和调整

检查生成的脚本：
- [ ] 角色定义完整且正确
- [ ] 对话格式符合规范
- [ ] 声音和方言分配合理
- [ ] 情感标签使用恰当
- [ ] 内容完整无遗漏

### 步骤 5：导入系统使用

1. 复制生成的脚本
2. 访问 SoulX-TTS Web 界面 (http://localhost:8000)
3. 切换到"🎙️ 多人播客"标签页
4. 粘贴脚本
5. 点击"🎙️ 生成播客"

## 🎯 使用技巧

### 技巧 1：分步骤处理

对于复杂的对话，可以分步骤：

**第 1 步：格式化**
```
请先格式化这段对话，只添加角色定义和对话标记，不要添加情感标签。
```

**第 2 步：添加情感**
```
现在请为上述脚本添加适当的情感标签。
```

**第 3 步：调整方言**
```
请将嘉宾的方言改为四川话。
```

### 技巧 2：提供上下文

给 AI 提供更多上下文信息，获得更好的结果：

```
请格式化以下对话，背景信息：
- 这是一个科技播客节目
- 主持人是女性，专业而友好
- 嘉宾是男性，是一位 AI 专家
- 对话氛围轻松愉快

对话内容：
[粘贴对话]
```

### 技巧 3：逐步优化

可以通过多轮对话不断优化：

```
用户：请生成脚本
AI：[生成初版脚本]

用户：请给主持人添加更多笑声标签
AI：[优化版脚本]

用户：嘉宾的语气可以更专业一些
AI：[最终版脚本]
```

### 技巧 4：保存常用提示

如果经常使用，可以：
1. 保存你最常用的提示词版本
2. 创建自定义的提示词模板
3. 添加你特有的需求和规则

## 🔧 常见问题

### Q1: AI 生成的格式不对怎么办？

**解决方案**:
- 确保提示词完整，包含格式说明
- 明确要求 AI 严格遵守格式
- 可以提供一个正确的示例让 AI 参考

**改进的提示**:
```
请严格按照以下格式生成脚本，不要有任何偏差：

角色定义格式（必须）：
@角色: 名称, 声音, 方言

对话格式（必须）：
[角色名称]: 对话内容

示例：
@角色: 主持人, 女声1, 普通话
[主持人]: 大家好。
```

### Q2: AI 添加了太多情感标签？

**解决方案**:
```
请适度使用情感标签，每段对话最多添加 1-2 个标签，并且只在非常合适的位置添加。
```

### Q3: AI 改变了对话内容？

**解决方案**:
```
请保持原有对话内容不变，只添加格式标记和必要的情感标签，不要修改或添加对话文本。
```

### Q4: 如何让 AI 理解特定的方言风格？

**解决方案**:
```
请为[地区]角色使用[方言]，特点如下：
- [特点 1]
- [特点 2]
- 示例用语：[示例词汇]
```

### Q5: 生成的脚本太长了？

**解决方案**:
```
请将这个长对话拆分为 3 个独立的播客脚本，每个脚本包含完整的角色定义和对话内容。
```

## 🌟 最佳实践

### 1. 提供清晰的对话内容

**好的输入**:
```
主持人：大家好，欢迎来到今天的节目。
嘉宾：你好，很高兴来到这里。
主持人：今天我们要聊聊人工智能。
```

**不好的输入**:
```
然后他说了什么来着...好像是欢迎...后面嘉宾说了一些话，具体忘了...
```

### 2. 明确你的需求

告诉 AI 你想要什么样的效果：
- 正式 vs 轻松
- 专业 vs 亲民
- 简洁 vs 详细

### 3. 逐步完善

不要期望一次就完美，可以：
1. 先生成基础版本
2. 然后添加情感
3. 最后调整细节

### 4. 保存好用的版本

当 AI 生成了一个特别好的脚本，可以：
- 保存作为示例
- 分析其中的优点
- 在下次使用时参考
