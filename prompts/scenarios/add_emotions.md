# 情感标签添加提示词

> 专门用于在已有播客脚本中添加副语言标签，增强情感表达

---

## 任务说明

你的任务是分析播客脚本的对话内容，在适当的位置添加副语言标签（如笑声、叹气等），使对话更加生动自然。

---

## 快速提示词

```
请为以下播客脚本添加适当的情感标签，使对话更生动：

可用标签：
- <|laughter|> 笑声 - 用于开心、幽默的时刻
- <|sigh|> 叹气 - 用于感慨、无奈的情绪
- <|breathing|> 呼吸声 - 用于思考、停顿
- <|coughing|> 咳嗽 - 用于打断、清嗓子

要求：
1. 根据对话情境判断情感
2. 在合适的位置插入标签
3. 不要过度使用，保持自然
4. 标签通常放在句中或句末

脚本内容：
[粘贴脚本]
```

---

## 副语言标签详解

### 1. 笑声 `<|laughter|>`

**使用场景**:
- 开心、愉快的时刻
- 讲笑话或有趣的内容
- 轻松、幽默的氛围
- 尴尬的缓解

**位置建议**:
- 句末：强调整句话的幽默感
- 句中：在幽默点之后立即添加

**示例**:
```
[主持人]: 这个问题问得好！<|laughter|> 让我来想想怎么回答。
[嘉宾]: 说实话，我当时也懵了。<|laughter|> 完全不知道该怎么办。
```

### 2. 叹气 `<|sigh|>`

**使用场景**:
- 感慨、感叹的情绪
- 无奈、遗憾的表达
- 深思熟虑的停顿
- 疲惫、释然的时刻

**位置建议**:
- 句首：铺垫情绪
- 句中：在关键词前后
- 独立使用：表示无言的感慨

**示例**:
```
[专家]: <|sigh|> 这个问题确实很复杂。
[创业者]: 创业这几年，<|sigh|> 真的不容易。
[听众]: 听完这个故事，我也不知道该说什么了。<|sigh|>
```

### 3. 呼吸声 `<|breathing|>`

**使用场景**:
- 思考和停顿
- 紧张、激动的情绪
- 深呼吸、平复心情
- 准备说重要内容之前

**位置建议**:
- 句首：表示停顿思考
- 句中：表示犹豫或酝酿
- 连续使用：表示深呼吸

**示例**:
```
[嘉宾]: <|breathing|> 让我想想该怎么说。这件事其实很复杂。
[主持人]: 接下来我要宣布一个重要消息。<|breathing|> 大家准备好了吗？
```

### 4. 咳嗽 `<|coughing|>`

**使用场景**:
- 清嗓子、准备发言
- 打断对方或话题
- 提醒、引起注意
- 掩饰尴尬

**位置建议**:
- 句首：清嗓子开场
- 独立使用：打断或提醒
- 句末：少用，可能表示不适

**示例**:
```
[嘉宾]: <|coughing|> 不好意思，我想说两句。
[主持人]: 那个... <|coughing|> 我们还是回到正题吧。
```

## 添加原则

### 原则 1：适度使用

**好的示例** (适度):
```
[主持人]: 今天的嘉宾很特别。<|laughter|> 他是我们的老朋友了。
[嘉宾]: 是啊，<|sigh|> 时间过得真快。
```

**不好的示例** (过度):
```
[主持人]: 今天的嘉宾<|laughter|>很特别。<|laughter|>他是<|laughter|>我们的老朋友了。<|laughter|>
```

### 原则 2：符合情境

标签必须符合对话的情感和语境：

**好的示例**:
```
[创业者]: 那段时间真的很艰难。<|sigh|> 但我们最终挺过来了。
```

**不好的示例**:
```
[创业者]: 那段时间真的很艰难。<|laughter|> 但我们最终挺过来了。
```
（艰难的经历不应该配笑声）

### 原则 3：自然流畅

标签的添加不应该影响对话的自然流畅：

**好的示例**:
```
[主持人]: 这个观点很新颖。<|laughter|> 我之前还真没想过。
```

**不好的示例**:
```
[主持人]: 这个<|laughter|>观点很<|laughter|>新颖。
```

## 情感分析指南

### 1. 识别情感基调

首先判断整段对话的情感基调：
- 轻松愉快 → 多用 `<|laughter|>`
- 深沉严肃 → 少用或不用情感标签
- 感慨回忆 → 适当使用 `<|sigh|>`
- 紧张激动 → 适当使用 `<|breathing|>`

### 2. 识别关键时刻

找出对话中的情感转折点：
- 从严肃到轻松 → 添加 `<|laughter|>`
- 从轻松到严肃 → 添加 `<|sigh|>` 或 `<|breathing|>`
- 话题转换 → 可以添加 `<|coughing|>` 作为过渡

### 3. 考虑角色性格

不同角色使用情感标签的频率不同：
- 活泼开朗的角色 → 多用 `<|laughter|>`
- 沉稳内敛的角色 → 少用情感标签
- 感性的角色 → 多用 `<|sigh|>`

## 实战练习

### 练习 1：添加笑声

**原文**:
```
[主持人]: 你知道吗，我第一次做播客的时候特别紧张。
[嘉宾]: 哈哈，我也是。还记得当时说错了好多话。
```

**优化**:
```
[主持人]: 你知道吗，我第一次做播客的时候特别紧张。<|laughter|>
[嘉宾]: 哈哈，我也是。<|laughter|> 还记得当时说错了好多话。
```

### 练习 2：添加叹气

**原文**:
```
[创业者]: 那段时间每天只睡四个小时。
[主持人]: 听起来真的很辛苦。
[创业者]: 是啊，不过现在回想起来，也值得了。
```

**优化**:
```
[创业者]: 那段时间每天只睡四个小时。<|sigh|>
[主持人]: 听起来真的很辛苦。
[创业者]: 是啊，<|sigh|> 不过现在回想起来，也值得了。
```

## 质量检查

添加情感标签后，检查：

- [ ] 标签使用适度，不过度
- [ ] 标签符合对话情境
- [ ] 标签位置合理，不影响阅读
- [ ] 不同情感标签搭配合理
- [ ] 整体效果自然流畅

---

## 使用建议

1. **先通读理解**: 先理解整段对话的情感和氛围
2. **标记关键点**: 找出需要强化情感的关键位置
3. **谨慎添加**: 宁少勿多，保持自然
4. **多次检查**: 添加后朗读一遍，感受是否自然
